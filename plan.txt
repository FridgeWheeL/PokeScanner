================================================================================
CURRENT CRITICAL ISSUES (June 2026)
================================================================================

1. "Scans forever" — OCR/API pipeline has NO timeouts or cancellation anywhere.
   - Task.WhenAll at line 528 waits forever if ANY HTTP request hangs
   - No CancellationTokenSource used anywhere
   - HttpClient default timeout is 100 seconds
   - _capture.Read() blocks indefinitely if camera driver hangs
   - Result: clicking Scan can lock up the app with no feedback

2. OCR unreliable even on simple English cards.
   - Tesseract with 3x upscale still produces garbage for clean photos
   - No preprocessing (CLAHE, binarization, denoising) before OCR
   - EngineMode.Default is slow; single tesseract Process() takes 1-5+ seconds
   - 4 sequential OCR passes per capture (1 top + up to 3 bottom fallbacks)
   - No char whitelist configured to constrain output

3. Capture loop design flaw:
   - Dispatcher.Invoke() on line 274 blocks the capture thread waiting for UI
   - If OCR or API work blocks the UI thread, capture stalls -> no new frames
   - Task.Delay(10).Wait() and Task.Delay(100).Wait() waste thread-pool threads

4. Full-card OCR fallback removed (was injecting attack/flavor text as card name)
   - But top crop alone often produces no valid name
   - localId+setTotal search is the only fallback when name OCR fails

5. Japanese card support untested.
   - Tesseract only has eng.traineddata loaded
   - TCGdex API supports Japanese but OCR path for Japanese text is missing

================================================================================
PROPOSED DIRECTION: SWITCH TO NEW OCR MODEL
================================================================================

Tesseract is proving too unreliable for this use case even with preprocessing.
Options to consider:

A) OCR.space API (free tier: 500 req/month, no API key needed for 25 req/day)
   - Cloud-based, better accuracy than Tesseract
   - Handles holo/foil cards better
   - Quick to integrate (HTTP POST with image)

B) Google Cloud Vision API
   - Best OCR accuracy
   - $1.50/1000 requests (pay-as-you-go, free tier available)
   - Requires API key setup

C) Windows OCR (Windows.Media.Ocr)
   - Built-in to Windows, no setup needed
   - Good for English text, runs locally
   - Windows-only (we're already WPF)

D) Microsoft Azure Computer Vision
   - Free tier: 5000 requests/month
   - Good accuracy, easy API

E) Gemini/Claude vision API
   - Can identify Pokemon directly (no API lookup needed)
   - Higher cost per request
   - Slower (network latency + processing)

================================================================================
NEW APPROACH (June 2026): Local LLM Vision Model
================================================================================

Tesseract OCR was unreliable — failed on simple English cards even with
aggressive preprocessing (CLAHE, sharpening, Otsu, upscale). Switched to
llava:13b vision model running locally via LiteLLM proxy at localhost:4000.

Setup:
- LiteLLM proxy running on this machine
- Model: pokemon-card-vision (llava:13b, supports_vision: true)
- Proxy URL: http://localhost:4000/chat/completions
- API key: LITELLM_MASTER_KEY in .env file (loaded at app runtime)

Current flow:
1. Capture ROI crop → encode as JPEG base64
2. Send to LLM: "Identify this Pokemon TCG card. Return only JSON..."
3. LLM returns {"name":"Bronzor","number":"071/094"}
4. Parse JSON → use name, number for TCGdex API lookup
5. If LLM fails → fallback to Tesseract + Otsu binarization

Implemented:
- LoadEnv() — reads .env for LITELLM_MASTER_KEY
- IdentifyWithLlmAsync() — POSTs to LiteLLM proxy, parses JSON response
- RunOcrAsync → tries LLM first, Tesseract fallback if needed
- HttpClient with 60s timeout

Key files:
- C:\Repos\PokeScanner\PokeScanner\PokeScanner\MainWindow.xaml.cs
  - LoadEnv() line ~48
  - IdentifyWithLlmAsync() line ~950
  - RunOcrAsync line ~392: LLM first, Tesseract fallback

Remaining issues:
1. [X] "Scans forever" — added CancellationTokenSource + cancellation tokens to all HTTP calls, SemaphoreSlim.WaitAsync, Task.WhenAll
2. Capture loop still uses blocking Dispatcher.Invoke
3. Need to test LLM accuracy with various card types (holo, foil, EX, Japanese)
4. LLM takes ~3-10s per request (local 13b model via Ollama/LiteLLM)
